Autenticación y Sesiones en PHP

Guía de Referencia

1. password_hash()

PHP 5.5+ provee una función que nos ayuda a generar 'hashes' seguros.

Algoritmo por default: bcrypt

Modo de uso:

$hash = password_hash($password, PASSWORD_DEFAULT);
  • El contenido de $hash es lo que almaceno en la base de datos.
  • PASSWORD_DEFAULT indica que algoritmo debe usar (el por defecto para passwords).
Nota: La función password_hash() genera automáticamente un salt aleatorio y lo incluye en el hash resultante, proporcionando una capa adicional de seguridad.

2. Logueo al Usuario

Usamos la SESSION para mantener al usuario logueado:

public function verifyUser() { $username = $_POST['username']; $password = $_POST['password']; $user = $this->model->getByUsername($username); // Encontró un user con el username que mandó, y tiene la algo contraseña if (!empty($user) && password_verify($password, $user->password)) { // INICIO LA SESSION Y LOGUEO AL USUARIO session_start(); $_SESSION['ID_USER'] = $user->id; $_SESSION['USERNAME'] = $user->username; header('Location: ver'); } else { $this->view->showLogin('Login incorrecto'); } }

Pasos del proceso de login:

  • Se reciben las credenciales desde el formulario POST
  • Se busca el usuario en la base de datos por su username
  • Se verifica la contraseña usando password_verify()
  • Si es correcto, se inicia la sesión y se guardan los datos del usuario
  • Se redirige a la página principal
  • Si falla, se muestra un mensaje de error

3. Usuario denegado

Usamos la SESSION para determinar si el usuario ya estaba logueado:

/** * Muestra la lista de tareas. */ public function showTasks() { // Barrera para usuario no logueado $this->checkLoggedIn(); // Obtengo tareas del model $tareas = $this->model->getAll(); // Se las paso a la vista $this->view->showTasks($tareas); } private function checkLoggedIn() { session_start(); if (!isset($_SESSION['ID_USER'])) { header('Location: ' . LOGIN); die(); } }
⚠️ IMPORTANTE: QUE LA SESSION ESTÉ INICIADA NO NECESARIAMENTE INDICA QUE EL USUARIO ESTÁ LOGUEADO

La función checkLoggedIn() actúa como una barrera de seguridad que:

  • Inicia la sesión si no está iniciada
  • Verifica que exista la variable $_SESSION['ID_USER']
  • Si no existe, redirige al login y detiene la ejecución
  • Si existe, permite continuar con la ejecución normal

4. Logout

Al desloguearse lo único que hay que hacer es borrar la información de la sesión.

session_destroy()

  • Elimina la sesión borrando todas las variables almacenadas.
  • Para eliminar la sesión siempre hay que iniciarla antes.
public function logout() { session_start(); session_destroy(); header('Location: ' . LOGIN); }

Proceso de logout:

  1. session_start() - Inicia/recupera la sesión actual
  2. session_destroy() - Destruye todos los datos asociados a la sesión
  3. header('Location: ' . LOGIN) - Redirige al usuario a la página de login
Buena práctica: Siempre redirigir al usuario después de hacer logout para evitar que intente acceder a páginas protegidas con la sesión destruida.

Resumen de Seguridad

Mejores prácticas implementadas:

  • Hash de contraseñas: Nunca almacenar contraseñas en texto plano
  • Uso de bcrypt: Algoritmo robusto y resistente a ataques de fuerza bruta
  • Verificación segura: Usar password_verify() para comparar contraseñas
  • Sesiones para autenticación: Mantener el estado de login del usuario
  • Validación en cada petición: Verificar autenticación antes de mostrar contenido protegido
  • Logout adecuado: Destruir la sesión completamente al cerrar sesión
